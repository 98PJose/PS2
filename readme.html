3. Module Documentation

3.1 copula_sim.py — Copula Simulation

Contains one function per copula type. All use the conditional (inverse CDF) method: given independent V1, V2 ~ U(0,1), set u2 = v2 and solve v1 = C(u1 | u2) for u1.

FunctionCopulaKey equation
sim_clayton(v1, v2, theta)Claytoneq 45: u1 = [1 + u2-θ (v1-θ/(1+θ) - 1)]-1/θ
sim_surv_clayton(v1, v2, theta)Survival Claytoneq 51: u1 = 1 - [(v̄1 v̄21+θ)-θ/(1+θ) - v̄2-θ + 1]-1/θ
sim_gaussian(v1, v2, rho)Gaussianeq 34: u1 = Φ(ρ Φ-1(v2) + √(1-ρ²) Φ-1(v1))
sim_student_t(v1, v2, rho, nu)Student teq 41: y1 = ρy2 + tν+1-1(v1)√[(1-ρ²)(ν+y2²)/(ν+1)]
sim_fgm(v1, v2, lam)FGMeq 53: u1 = [(1+ψ) - √((1+ψ)² - 4v1ψ)] / (2ψ), ψ = λ(1-2v2)
sim_mixture(...)Mixture of two copulasBernoulli selection per draw

Example:

import numpy as np
from exercise1.copula_sim import sim_gaussian

np.random.seed(42)
v1, v2 = np.random.rand(10000), np.random.rand(10000)
u1, u2 = sim_gaussian(v1, v2, rho=-0.7)
# u1, u2 are now dependent U(0,1) with Gaussian copula structure

3.2 copula_likelihood.py — Log-Likelihoods

Provides the log-likelihood of the copula density for each observation, summed over the sample.

Gaussian copula LL (eq 31):
ln cG = -½ ln(1-ρ²) - [ρ²(x₁²+x₂²) - 2ρx₁x₂] / [2(1-ρ²)]
where xi = Φ-1(ui).
Student t copula LL (eq 38):
ln ct = ln Γ((ν+2)/2) + ln Γ(ν/2) - 2 ln Γ((ν+1)/2) - ½ ln(1-ρ²) - (ν+2)/2 · ln(1 + Q/[ν(1-ρ²)]) + (ν+1)/2 · [ln(1+y₁²/ν) + ln(1+y₂²/ν)]
where Q = y₁² - 2ρy₁y₂ + y₂² and yi = tν-1(ui).

3.3 estimation.py — EDF & IFM Estimation

EDF (Empirical Distribution Function): Uses the Weibull plotting position F̂(x(k)) = k / (T+1), avoiding boundary values 0 and 1.

IFM method (Joe, 1997) in two steps:

  1. Step I: Estimate marginals via EDF → pseudo-observations ûi.
  2. Step II: Maximize copula log-likelihood given ûi to estimate copula parameters.

When EDF is used for Step I, this is also called the CML (Canonical Maximum Likelihood) method (slide 84).

FunctionDescriptionOptimizer
edf(x)Empirical CDF: rank/(T+1)—
estimate_gaussian_copula(u1, u2)Estimate ρ for Gaussian copulaBrent (bounded)
estimate_t_copula(u1, u2)Estimate (ρ, ν) for t copulaNelder-Mead

Example:

from exercise1.estimation import edf, estimate_gaussian_copula

# Given return observations r1, r2
u1_hat = edf(r1)
u2_hat = edf(r2)
result = estimate_gaussian_copula(u1_hat, u2_hat)
print(result["rho"])  # estimated correlation

5. Exercise Scenarios

5.1 Part 1.a — Copula Simulation & Dependence

Generates 100,000 bivariate dependent U(0,1) draws for 6 copulas. Produces scatter plots showing different dependence structures (lower-tail, upper-tail, elliptical, weak, independence). Reports Pearson's ρ, Spearman's ρS, and Kendall's τ.

5.2 Part 1.b — Portfolio Analysis

Transforms copula draws to returns (R1 ~ N(0,1), R2 ~ t(6)) and constructs portfolio rp = 0.4·r1 + 0.6·r2. Reports mean, std, skewness, kurtosis, VaR(1%), VaR(5%). Produces histograms with Normal PDF overlay to highlight tail differences. Reports the 6×6 correlation matrix of portfolio returns across copula models.

5.3 Part 1.c — IFM Estimation of Gaussian Copula

Uses simulated returns from the Gaussian copula (ρ = -0.7) as "observed data". Estimates marginals via EDF, then maximizes Gaussian copula LL to recover ρ. Validates that the estimated ρ is close to the true value.

5.4 Part 1.d — Mixture Copula + t Copula Estimation

Simulates from a mixture copula: 0.6·Clayton(10) + 0.4·SurvClayton(5). Both marginals are N(0,1). Estimates a t copula via IFM+EDF, recovering (ρ, ν). Also estimates a Gaussian copula for comparison. The t copula should achieve a higher LL due to its ability to capture tail dependence via low ν.

Note on mixture simulation: For each draw, a Bernoulli(0.6) random variable selects whether to draw from Clayton or Survival Clayton. Both components are evaluated on the same independent uniforms; the indicator selects which output to keep.

7.3 garch.py — GARCH and GJR Estimation (2.2b, 2.2c)

Estimates univariate volatility models by maximum likelihood with zero mean and Normal innovations.

GARCH(1,1) (eq 67):
ht = ω + α r2t-1 + β ht-1

Constraints: ω > 0, α ≥ 0, β ≥ 0, α + β < 1.
Unconditional variance: σ2 = ω / (1 - α - β).

Log-likelihood: LL = -½ Σt [ln(ht) + rt2/ht]
GJR(1,1) (eq 71):
ht = ω + (α + γ Dt-1) r2t-1 + β ht-1

where Dt-1 = 1 if rt-1 < 0 (leverage indicator).
Constraints: ω > 0, α ≥ 0, β ≥ 0, γ ≥ 0, α + β + γ/2 < 1.

The γ parameter captures the asymmetric leverage effect: negative shocks increase volatility more than positive shocks of equal magnitude.
FunctionDescriptionOptimizer
estimate_garch(r)GARCH(1,1) MLE, returns ω, α, β, ht, ztL-BFGS-B
estimate_gjr(r)GJR(1,1) MLE, returns ω, α, β, γ, ht, ztL-BFGS-B

Example:

from exercise2.garch import estimate_garch
result = estimate_garch(returns["BTC"].values)
print(result["alpha"], result["beta"])  # GARCH params
z = result["z"]  # standardized residuals z_t = r_t / sqrt(h_t)

7.4 dcc.py — DCC Model (2.2–2.4)

Implements the Dynamic Conditional Correlation model under both Gaussian and Student t copulas.

DCC dynamics (eq 69):
Qt = (1 - a - b) Ω̂ + a zt-1 z't-1 + b Qt-1

where Ω̂ is the sample correlation of zt, and Q1 = Ω̂.

Normalization (eq 68):
Ψt = Λt-1/2 Qt Λt-1/2
where Λt = diag(q11,t, ..., qnn,t).
This ensures Ψt is a valid correlation matrix with unit diagonal.
Gaussian copula DCC log-likelihood (eq 72):
LLG = -½ Σt [ln|Ψt| + η't (Ψt-1 - In) ηt]
Student t copula DCC log-likelihood (eq 76):
LLt = T/2 ln A(v) - ½ Σt ln|Ψt| - (v+n)/2 Σt ln[1 + v-1 η't Ψt-1 ηt] + (v+1)/2 Σt Σi ln[1 + v-1 η2it]

where A(v) = Γ((v+n)/2) Γ(v/2)n-1 / Γ((v+1)/2)n.
FunctionDescriptionParameters estimated
estimate_dcc_gaussian(z)Gaussian copula DCCa, b
estimate_dcc_t(u)Student t copula DCCa, b, ν

8. Exercise 2 Scenarios

8.1 Part 2.1 — Descriptive Statistics

Reports mean, std, skewness, kurtosis for BTC, ETH, ADA daily returns, plus the 3×3 sample correlation matrix. All three series show excess kurtosis (leptokurtic) and negative skewness (especially BTC).

8.2 Part 2.2 — DCC Estimation under Three Marginal Models

The IFM method estimates the DCC in two steps:

  1. Step I — Marginals:
    • 2.2a (EDF): uit = rank(rit)/(T+1), then zit = Φ-1(uit). This is a static, nonparametric transformation that does not capture time-varying volatility.
    • 2.2b (GARCH-N): Estimate GARCH(1,1) per series, then zit = rit/√hit. Captures volatility clustering.
    • 2.2c (GJR-N): Same as GARCH but with leverage term. Captures asymmetric volatility response.
  2. Step II — DCC: Estimate (a, b) by maximizing the Gaussian copula log-likelihood given the standardized residuals from Step I.
Key observation from 2.3: The EDF-based DCC produces smoother dynamic correlations because the EDF is a static transformation that cannot capture time-varying volatility clustering. The GARCH and GJR models produce more volatile correlation paths because the standardized residuals zt = rt/√ht remove conditional heteroscedasticity, allowing the DCC to isolate pure correlation dynamics. GARCH and GJR results are similar, with GJR showing slightly different patterns due to the leverage effect.

8.3 Part 2.4 — DCC under Student t Copula

Estimates (a, b, ν) under the t copula with EDF marginals. The t copula accounts for tail dependence via the degrees-of-freedom parameter ν. Lower ν implies heavier joint tails. The t copula LL should be at least as high as the Gaussian copula LL (the Gaussian is the limiting case ν → ∞).

10. Mathematical Setup

The bivariate Normal Mixture (NM) copula is a mixture of two Gaussian copulas sharing the same uniform margins but differing in correlation:

CNM(u1, u2) = π · CG(u1, u2; ρ1) + (1 − π) · CG(u1, u2; ρ2),   π ∈ (0, 1).

Because differentiation is linear, the conditional CDF of U1 given U2 = u2 is the weighted mixture of the two Gaussian conditional CDFs (problem statement, Exercise 3):

CNM(u1 | u2) = π · Φ&left;((Φ−1(u1) − ρ1 Φ−1(u2)) / √(1 − ρ12)&right;) + (1 − π) · Φ&left;((Φ−1(u1) − ρ2 Φ−1(u2)) / √(1 − ρ22)&right;).

A q-quantile curve is the locus of points (u1, u2) satisfying CNM(u1 | u2) = q, for a fixed probability q ∈ (0, 1). Following Section 9 of the course slides, u2 is placed on the x-axis and u1 is solved for numerically on the y-axis. Because the right-hand side is a sum of two Phi terms (not a single Phi), there is no closed-form inverse — scipy.optimize.brentq is used instead. The conditional CDF is strictly increasing in u1 (each Phi is increasing and both mixture weights are positive), so a unique root exists for any q in (0, 1).

Note on notation. The problem text phrases the inversion as "back out a value of u2 from CNM(u1 | u2) = q". Taken literally with π = 0.3, ρ1 = −0.7, ρ2 = 0.4, the map u2 ↦ CNM(u1 | u2) is not monotone onto (0, 1); its limiting values at u2 → 0 and u2 → 1 are 0.7 and 0.3 respectively, so q ∈ {0.05, 0.25, 0.75, 0.95} would have no solution. Inverting in u1 is strictly monotone, delivers a solution for every q in (0, 1), and matches the convention used for all other copulas in the slides. This is the convention adopted here.
